# Sudoku

> Think. Scan. Place.

A feature-rich Sudoku app for iOS and Android built with React Native and Expo. Four difficulty levels, score tracking, statistics, haptic feedback, sound effects, and full dark/light theme support — all persisted locally with SQLite.

## Features

- **4 difficulty levels** — Easy, Medium, Hard, Expert (54%–70% cells removed)
- **Unique puzzle guarantee** — every generated board has exactly one solution
- **Scoring system** — base points × factor, row/col/grid completion bonuses, error penalties
- **Clue system** — spend points to reveal a cell (limited per game)
- **Score multiplier** — factor decreases every 30 seconds, rewarding fast solves
- **Resume game** — save progress and continue later
- **Statistics** — global stats and per-level breakdown (best time, max points, streak)
- **Dark / light theme** — system-aware with manual override, persisted to DB
- **Haptic feedback** — two intensities, user toggle
- **Sound effects** — on cell interaction
- **Animations** — cell spin on row/col/grid/number completion via Reanimated

## Architecture decisions

**Zustand over Redux** — the state shape is simple (board, scores, notifications) and doesn't need reducers or middleware. Three small stores keep related state co-located without boilerplate.

**SQLite + Drizzle ORM over AsyncStorage** — game stats need relational queries (per-level aggregates, streaks, time totals). Drizzle gives type-safe schema + migrations that run automatically on startup, so the DB is always in sync with the schema.

**CNG (Continuous Native Generation)** — native `android/` and `ios/` folders are gitignored and regenerated by Expo Prebuild on each build. All native config lives in `app.json`, keeping the repo clean and EAS Build in sync.

**Solution-based validation** — when a player places a number, it's compared against the pre-generated solution rather than re-checking board rules. This is both faster and correct: it prevents wrong placements that happen to not conflict with the current board state.

**Hooks for all side effects** — `useHaptic`, `useLoadSound`, `useTimer`, `useLevel`, `useGameBoard` — logic is extracted from components and independently testable.

## Project structure

```
app/          # Screens (Expo Router file-based routing)
components/   # Reusable UI — grid, stats, settings, shared
hooks/        # Custom hooks — haptic, sound, timer, styles, game logic
store/        # Zustand stores, Drizzle schema, DB services
constants/    # Colors, dimensions, levels, typography
utils/        # Sudoku algorithms, formatters, DB backup
assets/       # Fonts, images, sounds, SVGs
```

## Getting started

### Prerequisites

- Node.js (LTS)
- Expo CLI — `npm install -g expo-cli`
- iOS Simulator (Mac) or Android Emulator

### Install

```bash
npm install
```

### Run

```bash
npm start          # Expo dev server
npm run ios        # iOS Simulator
npm run android    # Android Emulator
```

### Test

```bash
npm test
```

### Lint

```bash
npm run lint
```

---

## Database

Schema is defined in `store/schema.ts` (5 tables: `settings`, `gameScores`, `globalStats`, `levelStats`, `savedGames`). Migrations run automatically on app start via Drizzle Kit.

```bash
npx drizzle-kit generate   # generate migration from schema changes
```
